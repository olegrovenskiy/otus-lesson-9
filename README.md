# otus-lesson-9

1. Принцип единственной ответственности;

Классы - Validator, Messager, Checker, Settings, RandomDigitGenerator  - несут в себе 
выполнение только одной функции.

2. Принцип открытости/закрытости;

Пример его реалиизации на классе - NumberGenerator, в котором для генерации например
рандомного числа используется RandomDigitGenerator. То есть если потребуется изменить способ генерации, 
например на случайные чётные, то не нужно будет менять код NumberGenerator, а просто добавить
новый класс RandomEvenDigitGenerator и использовать его в NumberGenerator

3. Принцип инверсии зависимостей;

На примере NumberGenerator зависимость от Generator заменена на интерфейс
IGenerator, котоый определяет метод GenerateNumber, реалтзованный в RandomDigitGenerator

4. Принцип разделения интерфейса;

Все интерфейсы требуют только минимально необходимый набор параметров, например 
IGenerator требует только диапозон генерируемого числа

5. Принцип подстановки Барбары Лисков

Класс генерации числа NumberGenerator, использует специализированный класс 
генератор - RandomDigitGenerator, в котором уже реализован метод рандомной генерации. 
Этот класс реализует базовый интерфейс IGenerator. Если мы захотим использовать 
другой способ генераци, то нам надо добавить новый класс с реализацией нового метода 
генерации - в моём примере это RandomDigitGeneratorLiskov, в которм для примера 
изменён метод генерации. Далее уже используется этот класс, и NumberGenerator не заметил
что уже использует другой класс - то есть возможность вместо базового типа подставить 
любой его подтип.

По факту этот пример также демонстрирует и принцип инверсии зависимостей.
